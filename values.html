Values

01

Build useful things.

not just impressive things. the best projects solve a real problem for a real person. if it doesn't help someone, i'm not interested.

02

Communicate clearly.

technical skill is only as useful as your ability to explain it. i work hard to make complex ideas feel simple — not simplified.

03

Stay curious.

i'd rather ask a "dumb" question than stay stuck with a wrong assumption. curiosity is how i learn and how i improve.

04

Make tech feel human.

software is for people. i want the things i build to feel intuitive, accessible, and genuinely useful — not just functional.

05

Keep improving.

i learn by doing. i'm never finished. every project teaches me something, and i try to apply it to the next one.

06

Think about the person using the product.

before i write a single line of code, i want to understand who will use it, what they need, and what might frustrate them. empathy is part of engineering.
